# ToDo

按日期管理的轻量级待办事项应用。纯前端实现，数据保存在浏览器本地，无需后端或账号。

## 功能特性

- **日期导航** — 左侧树形结构按年 / 月 / 日浏览，支持自定义日期选择器快速添加日期
- **自动结转** — 新建日期时，自动将上一日未完成的待办带入（已完成项不结转）
- **计划分类** — 支持「默认计划」与「计划外」两类，分别展示与编号
- **状态管理** — 未完成 / 完成 / 搁置三种状态，卡片颜色随状态变化
- **备注** — 每条待办可添加备注，输入框随内容自动增高
- **往期只读** — 早于今天的日期仅可查看，不可编辑
- **导出 / 导入** — JSON 格式备份与恢复，导入时与现有数据合并
- **快捷复制** — 一键生成「今日计划」或「日报」并复制到剪贴板
- **主题切换** — 深色 / 浅色模式，偏好保存在本地
- **动效与适配** — GSAP 卡片动画、粒子背景；支持移动端侧栏与响应式布局
- **无障碍** — 尊重 `prefers-reduced-motion`，减少动画时自动降级

## 快速开始

无需安装依赖，克隆或下载后直接打开即可：

```bash
# 方式一：直接打开
# 在浏览器中打开 todo.html

# 方式二：本地静态服务（推荐，Worker 导入等功能更稳定）
npx serve .
# 或
python -m http.server 8080
```

然后访问 `http://localhost:8080/todo.html`（端口以实际为准）。

> 通过 `file://` 协议打开时，部分浏览器可能限制 Web Worker；若导入大文件较慢，建议使用本地静态服务。

## 使用说明

### 添加日期

1. 在左侧日期选择器中选择日期
2. 点击「添加」
3. 若该日期尚无记录，系统会从最近一个有数据的日期带入未完成待办

### 管理待办

1. 在左侧日期树中点击某一天
2. 在顶部输入框填写内容，选择「默认计划」或「计划外」，点击「添加」
3. 点击卡片上的状态按钮切换完成 / 搁置 / 未完成
4. 在卡片底部备注区填写补充说明
5. 点击右上角 `×` 删除待办

### 导出与导入

- **导出 JSON** — 下载完整备份文件（含所有日期与当前选中日期）
- **导入 JSON** — 选择备份文件，数据会**合并**到现有记录中（不会清空本地数据）

### 今日计划与日报

选中某日期后：

- **今日计划** — 复制当日「默认计划」项列表（格式：`序号.内容`）
- **生成日报** — 复制结构化日报，含状态与备注；计划外事项单独列出

### 主题

左下角开关可在深色 / 浅色模式间切换，设置保存在 `localStorage`。

## 数据存储

| 项 | 说明 |
|---|---|
| 存储位置 | 浏览器 `localStorage` |
| 存储键 | `todo-app-data` |
| 主题键 | `todo-theme`（`light` / 默认深色） |

### 数据结构

```json
{
  "selectedDate": "2026-07-07",
  "dates": {
    "2026-07-07": [
      {
        "id": "abc123",
        "text": "完成需求文档",
        "status": "pending",
        "plan": "default",
        "note": "下午三点前提交",
        "order": 1
      }
    ]
  }
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 唯一标识 |
| `text` | string | 待办内容 |
| `status` | string | `pending` / `completed` / `on_hold` |
| `plan` | string | `default`（默认计划）或 `unplanned`（计划外） |
| `note` | string | 备注，可为空 |
| `order` | number | 同计划类型内的排序序号 |

导出文件在以上结构外额外包含 `version` 与 `exportedAt` 字段。
